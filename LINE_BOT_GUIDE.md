# LINE 機器人輸入說明書

> 適用對象：廚房現場備料人員（直接在 LINE 輸入數據）
> 版本：2026-04

---

## 基本規則（必讀）

在開始輸入前，請牢記以下五點，**任何一點違反都會讓整批數據被拒絕**：

| # | 規則 | 正確 | 錯誤 |
| :---: | :--- | :--- | :--- |
| 1 | 重量只填數字，不加單位 | `3000` | `3000g` ← 會造成報表錯誤 |
| 2 | 欄位之間只用 `/` 分隔 | `節瓜/3000` | `節瓜 3000` 或 `節瓜,3000` |
| 3 | 品項名稱必須與白名單完全一致 | `牛小排` | `和牛` 或 `牛小 排`（多空格） |
| 4 | 日期固定 8 位數字 | `20260411` | `2026/04/11` 或 `04/11` |
| 5 | 重量必須符合 進貨 ≥ 退冰 ≥ 成品 | 進貨`10000` 退冰`9600` 成品`9000` | 進貨`8000` 退冰`9600`（退冰比進貨重） |

## 新增紀錄

### 無退冰流程（5 段式）— 適用蔬果類

```
日期/品項/進貨重量/成品重量/負責人
```

**實際範例：**
```
20260411/節瓜/3000/2000/哲維
```

可以一次輸入多筆，每行一筆：
```
20260411/節瓜/3000/2000/哲維
20260411/西芹/5000/4200/哲維
20260411/洋蔥/2000/1800/哲維
```

---

### 有退冰流程（6 段式）— 適用冷凍肉類

```
日期/品項/進貨重量/退冰重量/成品重量/負責人
```

**實際範例：**
```
20260411/牛小排/10000/9600/9000/哲維
```

---

### 混合輸入（同時有退冰和無退冰）

可以直接混在一起，系統根據段數自動判斷：

```
20260411/牛小排/10000/9600/9000/哲維
20260411/節瓜/3000/2000/哲維
20260411/雞肝/5000/4800/4500/哲維
```

> 一次最多 **10 筆**，超過請分兩次傳送。

## 修改已有紀錄

若上傳數據有誤，請用「修改」模式。行號請查看 Google 試算表最左側的列編號。

### 無退冰（7 段式）

```
修改/行號/日期/品項/進貨重量/成品重量/負責人
```

**實際範例：**
```
修改/23/20260411/節瓜/3000/1800/哲維
```

### 有退冰（8 段式）

```
修改/行號/日期/品項/進貨重量/退冰重量/成品重量/負責人
```

**實際範例：**
```
修改/15/20260411/牛小排/10000/9500/8800/哲維
```

## 機器人回覆說明

### 成功

```
✅ 數據處理完成
- 狀態：全數入庫成功
- 筆數：3 筆
- 分頁：「總數據」已更新
```

### 失敗（系統會說明第幾筆、錯在哪）

| 機器人回覆 | 原因 | 修正方法 |
| :--- | :--- | :--- |
| `❌ 第N筆格式錯誤（段數不足）` | 斜線數量不夠 | 檢查是否漏填某欄位 |
| `❌ 第N筆錯誤：品項「XXX」不在名單內` | 品項名稱打錯 | 確認試算表白名單的標準名稱 |
| `❌ 第N筆數據邏輯錯誤：成品重量不可大於進貨重量` | 成品填超過進貨 | 重新核對秤重數值 |
| `❌ 第N筆數據邏輯錯誤：重量必須符合「進貨 ≥ 退冰 ≥ 成品」` | 退冰或成品數值異常 | 重新核對三欄數值 |
| `❌ 第N筆錯誤：重量欄位必須為數字` | 重量欄位混入文字或單位 | 只填數字，不加 g、kg 等單位 |
| `⚠️ 權限錯誤：此機器人僅限指定廚房管理群組使用` | 從錯誤群組或私訊傳送 | 確認在正確的 LINE 群組內傳送 |

> ❗ 任何一筆出錯，**整批都不會寫入**，修正後需要重新傳送全部。
