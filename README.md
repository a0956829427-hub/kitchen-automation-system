# 廚房備料自動化管理系統 (Kitchen Automation System)

這是一套為專業廚房設計的輕量化管理系統，旨在透過行動端介面簡化每日的「叫貨統計」與「備料損耗紀錄」，並透過自動化流程將數據同步至 Google 試算表與 Notion 作品集。

---

## 💻 前端：系統功能分頁說明

### 1. 叫貨單 (Order)
用於每日向供應商叫貨前的數量統計。
* **填寫方式**：在品項右側直接輸入數量。
* **自動縮寫**：系統會根據預設單位自動生成對應縮寫。
* **確認送出**：點擊後自動跳轉至「匯出」頁面產出 LINE 格式純文字。
* **快速清空**：點擊「✕ 清空」可一鍵重置所有暫存數據。

### 2. 損耗回報 (Loss Report)
用於精確紀錄食材處理過程中的重量損失（出成率計算）。
* **核心欄位**：
    * **日期/負責人**：系統自動記憶姓名，日期請維持 8 位數格式（如 `20260411`）。
    * **三欄式數據**：
        1. **進貨(g)**：原始採購重量。
        2. **退冰(g)**：退冰後重量（蔬果不需退冰可留空）。
        3. **成品(g)**：最終可使用重量（系統依此計算出成率）。

### 3. 匯出 (Export)
將數據轉換為標準化文字，以便快速貼上至 LINE 工作群組。
* **自動分批處理**：為避免數據過長導致系統處理過載，若單次紀錄**超過 10 筆**，系統會自動拆分為「第 1 批」、「第 2 批」按鈕分次匯出。
* **標準格式**：叫貨為 `節瓜-5根` 格式；損耗使用 `/` 分隔格式，便於後端解析。

### 4. 設定 (Settings)
自定義專屬於該廚房的食材名單。
* **品項管理**：可隨時新增類別、修改縮寫或刪除不常用的品項。
* **本地儲存機制**：資料存放於手機瀏覽器的 LocalStorage。*(注意：若清除瀏覽器快取或使用無痕模式，自訂紀錄將會遺失)*

## 🤖 後端：LINE 機器人使用與輸入規範

本系統透過 LINE 機器人接收數據，並設有**雙重校驗機制**（群組權限驗證、品項白名單比對），任一驗證失敗即取消寫入並回報錯誤。

### ⚠️ 核心輸入準則（四大禁令）
1. **嚴禁輸入單位**：重量請輸入純數字（例：`3000`）。若輸入 `3000g` 會導致報表顯示 `#VALUE!`。
2. **符號唯一性**：各欄位之間僅使用 `/`（斜線）分隔，不可使用空格或逗號。
3. **品項名稱一致**：輸入品項必須與白名單完全一致（例：名單為「牛小排」，輸入「和牛」會報錯）。
4. **日期格式固定**：統一使用 8 位數字（如 `20260411`）。

### 📝 新增備料紀錄（5 或 6 段式）
系統根據輸入的「段數」自動判斷有無退冰流程：
* **無退冰流程 (5 段式)**：適用蔬果。
  * `日期/品項/進貨重量/成品重量/負責人` *(範例：20260411/節瓜/3000/2000/哲維)*
* **有退冰流程 (6 段式)**：適用冷凍肉類。
  * `日期/品項/進貨重量/退冰重量/成品重量/負責人` *(範例：20260411/牛小排/10000/9600/9000/哲維)*

### 🔄 修改已有紀錄（7 或 8 段式）
若上傳有誤需修正，請使用修改模式（行號請查看試算表最左側）：
* `修改/行號/日期/品項/進貨重量/[退冰重量]/成品重量/負責人`
  *(範例：修改/23/20260411/節瓜/3000/1800/哲維)*

### 💬 機器人回覆與常見錯誤排除
傳送後請確認回覆狀態：
* **✅ 上傳成功**：數據已準確寫入。
* **❌ 錯誤訊息**：請對照下表修正後重新發送：

| 錯誤現象 / 回覆 | 可能原因 | 修正方法 |
| :--- | :--- | :--- |
| **報表顯示 #VALUE!** | 輸入了字母（如 `3000g`） | 刪除字母，僅保留數字。 |
| **品項不在名單內** | 名稱打錯、簡繁差異、多打空格 | 檢查「各類名單」中的標準名稱。 |
| **格式錯誤** | 斜線數量不足 | 檢查是否漏打某個重量或負責人姓名。 |
| **數據位移** | 多打或少打一個斜線 | 重新計算斜線數量（5 段或 6 段）。 |

## 📊 資料庫結構 (Data Schema) 與報表邏輯

本系統不僅實現自動化寫入，更透過 Google Sheets 強大的數據處理功能，將原始數據轉化為可視化的備料分析。

### 1. 總數據 (RawData)
* **角色**：存放所有 LINE 傳送過來的原始 JSON 解析數據。
* **關鍵維度**：包含進貨、退冰、成品三階段重量，作為計算「出成率」的基準。

### 2. 各類名單 (MasterList)
* **角色**：系統白名單與防錯機制。
* **功能**：連動 n8n 進行品項校驗，確保進入資料庫的品項名稱完全統一，避免統計錯誤。

### 3. 結算報表 (Analytics Report) - **核心精華**
這是本系統最具分析價值的部分，透過公式自動產出月度/年度結算。
* **動態彙整**：使用 `QUERY` 函數自動依據月份與品項進行彙整。
* **自動計算**：
    * **年總進貨/成品重量**：協助廚房掌握採購量級。
    * **平均損耗比 (%)**：關鍵分析指標。例如「四季豆」損耗比 31.66%、「山蘿蔔葉」損耗比 73.33%，能直接協助行政主廚進行成本核算與菜單調整。

> **技術亮點**：
> 報表核心邏輯：`=IFERROR(QUERY('總數據'!A:J, "SELECT B, SUM(C), SUM(D), SUM(F), AVG(H) ... GROUP BY B"), "")`

## 📦 快速部署指南

若欲在其他環境部署本系統，請遵循以下步驟：

### 1. 匯入 n8n Workflow
1. 開啟 n8n 實例，選擇 **Workflows → Import**。
2. 上傳專案中的 `clean-workflow.json`（已去敏化版本）。
3. 在 n8n 的 **Credentials** 管理介面設定以下憑證：
   * `LINE_CHANNEL_ACCESS_TOKEN`: 來自 LINE Developers。
   * `GOOGLE_SHEETS_OAUTH2`: 授權存取指定試算表。

### 2. 環境變數設定
請於 n8n 全域變數或節點內填入以下必要資訊：
* `GROUP_ID`: 授權執行指令的指定 LINE 群組。
* `SPREADSHEET_ID`: 目標 Google Sheets 的唯一代碼。

### 3. Google Sheets 結構要求
確保試算表包含以下分頁：
* **總數據**: 欄位包含 `日期`, `品項`, `進貨重量(g)`, `退冰實際重量`, `成品重量(g)`, `負責人`。
* **各類名單**: 單一欄位 A 供系統讀取作為白名單比對。

## 🛡️ 安全注意事項與維護

* **數據脫敏**: 本倉庫不包含任何 API Key、Token 或實體試算表連結。
* **安全性驗證**: 系統設有 Group ID 鎖定機制，非授權群組傳送之指令將被 Webhook 直接拋棄。
* **維護建議**: 建議每月定期檢查 `LocalStorage` 佔用量，若品項更新頻繁，可手動執行「資料清理」確保載入速度。
